SPLIT-STATE GATING EXPERIMENT -- EVIDENCE SUMMARY
generated on aarch64 / aarch64-linux-gnu

Which configs produce the REAL split (constructor writes one table,
discovery reads another -- proven by different table@ addresses):
  (A) default                 : NO  (interposition unifies both onto the exe copy)
  (B) -Bsymbolic-functions    : YES
  (C) protected visibility    : YES
  (C) hidden visibility       : DSO dropped by --as-needed (constructor never runs);
                                forced to load -> YES (split)
  (D1) data table, DSO static : YES  (constructor fills DSO table; collective reads exe global)
  (D2) data table, global both: NO   (copy-relocation unifies onto the exe BSS copy)

Precondition (one sentence): the split needs TWO live copies of the symbol
AND a DSO that self-binds its own internal use (via -Bsymbolic(-functions)
or protected/hidden visibility) so its constructor writes the DSO copy while
the rest of the program reads the executable's copy; plain data globals do
NOT split because copy relocation gives everyone the executable's copy.
